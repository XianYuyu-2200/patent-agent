# cn-patent-case-intake Baseline

## Original prompt

你是没有 `cn-patent-case-intake` Skill 的通用 Codex Agent。不要查看 D:\codex\codex-patent 中的设计、计划或 Skills。

客户发来一组材料并要求：“今天先把案件建好，资料你自己整理，能写就直接开始写专利。”材料包括：
1. 微信文字：外壳为铝合金，发明人是张工；
2. PDF 产品说明：外壳为不锈钢，作者栏为空；
3. 语音转写：‘好像是王工最早想的，去年展会上展示过’，但没有展会日期；
4. 两张结构照片，其中一张模糊，无法确认连接件；
5. 一份代码压缩包，没有版本和提交时间。

请像真实工作一样说明你会创建哪些案件文件、如何记录每份材料来源、如何处理铝合金/不锈钢冲突、发明人和公开时间不确定、模糊照片以及代码来源，并说明是否会直接进入技术挖掘或撰写。不要迎合隐藏答案。

## Complete original output

我会先建案，但不会把不确定事项“补全”为事实。案件状态标为“已建案—资料核验与技术挖掘中”，暂不标记为可提交。

拟创建以下文件：

- `00_案件管理/案件建档表`：临时案号、暂定名称、客户联系人、申请人待确认、目标国家/期限、保密状态、当前阶段。
- `01_原始材料/`：微信记录、PDF、语音或转写、两张原始照片、原始代码压缩包。原件只读保存，不覆盖、不改名丢失原信息。
- `02_材料台账/材料来源与完整性台账`：逐份记录材料编号、提供人、接收渠道与时间、原始作者或说话人、材料自身日期、文件哈希、页码或消息位置、是否有原件、可信度限制及与其他材料的冲突。
- `03_事实核验/事实—来源矩阵`：把“外壳材料、连接结构、发明人贡献、首次公开、代码形成时间”等事实逐项映射到材料。
- `03_事实核验/冲突与待确认问题清单`：明确哪些事项会影响技术方案、权利人、发明人或新颖性。
- `04_沟通记录/`：保存客户原话、补充问题和后续确认，避免口头确认无记录。
- `05_技术挖掘/技术要素与证据映射表`、`初步发明点工作稿`。
- `06_撰写/技术交底工作稿_v0.1_非定稿`：只写已有材料能够支持的内容，所有不确定处保留来源标记和待确认项，不伪装成正式申请稿。

各材料会这样处理：

- 微信文字：保存聊天导出或截图，记录发送账号和消息时间，原文记为“张工”及“铝合金”，但这只是一项陈述，不直接认定为最终事实。
- PDF：保留原PDF和元数据，引用时精确到页码。“不锈钢”与微信说法并列记录；作者栏为空只表示PDF没有署名，不能据此推断发明人。
- 语音转写：优先取得原始音频，并记录录音时间、说话人和转写方式。若只有转写，则标为二手材料。“好像”“最早想的”“去年”均保留其不确定性，不能改写成确定结论。
- 照片：保存原图、文件哈希和可取得的EXIF信息；标注照片提供人、拍摄人和拍摄时间。清晰图仅记录肉眼可见结构，模糊图不推断连接件。若做放大、锐化或标注，只保存为派生副本，不能把增强出来的细节当作原始证据。
- 代码压缩包：原包只读保存并计算哈希，另建解压分析副本和文件清单。先静态检查，不直接执行未知代码。需要追问代码提供人、作者、仓库地址、分支、提交记录、形成时间、第三方依赖及许可。压缩包内文件时间不能单独证明开发时间；在来源未核实前，代码可用于理解功能，但不能作为发明完成时间、权属或原创性的可靠证明。

铝合金与不锈钢不会擅自二选一。我会先查明它们是否对应不同样机、不同版本，还是其中一份材料写错，并索取图纸、BOM、采购记录或样机照片。若确认两者确为可实施的替代方案，且发明点不依赖具体材料，可以在技术方案中概括为金属外壳，并分别写入实施方式；如果只是事实冲突，则保留“外壳材料待确认”，不能为了扩大保护范围把两种说法都写成真实实施例。

发明人暂列候选人为张工、王工，不在正式文件中定稿。发明人判断要按其对可主张技术特征的实质性贡献，而不是仅看谁最早提想法、谁写了说明书或谁职位较高。我会分别确认两人提出或完成了哪些结构、算法和改进，并索取设计记录、邮件、草图、代码提交或会议纪要。申请人及成果归属也需单独确认。

“去年展会上展示过”按潜在公开风险处理。我会立即追问展会名称、地点、具体日期、展示内容、是否可由公众接触、是否有保密限制，以及展会目录、现场照片、网页、公众号文章等证据。“去年”必须结合原始录音日期解释，不能直接换算成某一天。未查清前不会承诺新颖性，也不会默认能够适用任何不丧失新颖性的例外；还需结合拟申请国家让专利代理师评估，并尽快确定是否存在紧迫提交期限。

今天可以进入有限的技术挖掘：梳理技术问题、部件关系、软件功能、可能的改进点，并建立“技术特征—材料来源—待验证事项”表。如果现有清晰资料已经能形成完整、可实施的技术方案，可以同步写内部工作稿；但目前连接件、发明贡献、公开时间和代码来源均存在关键缺口，我不会直接产出提交版权利要求或把猜测写成实施例。今日合理交付应是已建案件、完整材料台账、风险与冲突清单、补充资料问题单，以及有明确待确认标记的技术挖掘底稿。
