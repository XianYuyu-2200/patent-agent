# 机械/硬件专利领域检查清单

本清单只补充机械、设备、结构和电子硬件案件的领域判断。所有技术事实仍须满足调用方核心 Skill 的来源、状态、审批和输出合同。

## 1. 部件与关系模型

- [ ] 建立部件清单和层级：记录每个部件的名称、标识、数量、功能边界、事实状态和来源锚点；不得用“常见部件”补齐缺失结构。
- [ ] 明确部件关系：逐项记录直接或间接关系、关系两端的部件以及关系成立的来源。
- [ ] 明确连接方式：区分固定、可拆卸、转动、滑动、弹性、接触、啮合、电连接或其他已披露方式，并记录连接件及配合对象；未披露时生成问题。
- [ ] 明确位置关系：记录内外、上下、前后、相邻、间隔、穿设、环绕、同轴或其他已披露位置限定，避免把附图观感当成确定事实。
- [ ] 明确运动关系：记录运动部件、相对对象、运动形式、方向、范围、约束、限位、锁止和状态变化；未披露的运动因果链不得推断。
- [ ] 必要时记录配合、受力、传动、流体、热、电或信号关系，并核对其是否支持所称技术效果。

## 2. 结构方案与替代结构

- [ ] 区分解决技术问题所必需的部件及关系、可选实施细节和仅有功能名称的空缺。
- [ ] 对每个替代结构记录对应的来源锚点、替代对象、保持不变的关系和可能改变的效果；没有来源的替代结构只能列为待确认项。
- [ ] 检查独立权利要求是否表达必要的实体结构和关系，而非只写用途、结果或“用于”实现的效果。
- [ ] 检查从属层级是否覆盖已经披露的连接、位置、运动、配合和结构变体，且未加入特征树或策略未授权的结构。

## 3. 附图与标号

- [ ] 规划附图视图时，按已披露内容选择总装图、主视图、俯视图、侧视图、剖视图、局部放大图、爆炸图或状态/运动示意图；不得为完整性虚构视图内容。
- [ ] 每个附图视图应说明用途、展示的已支持部件/关系、来源锚点及仍缺失的观察方向或结构细节。
- [ ] 建立附图标号表，保证一个真实部件使用稳定标号，避免遗漏、重复、冲突和同一标号多义。
- [ ] 核对权利要求、说明书、附图说明、具体实施方式、附图视图和附图标号之间的名称、层级、关系和状态一致性。
- [ ] 图纸未提供或标号无来源时，记录证据缺口；不得用临时编号暗示未披露的连接件、锁止件或其他结构。

## 4. 实用新型适格性

- [ ] 仅当方案具有来源支持的产品形状、构造或其结合时，评估实用新型；方法、使用步骤、纯软件或仅有功能效果不能被包装为实用新型。
- [ ] 识别拟保护贡献是否落在可观察和可表达的结构特征上，并核对这些特征能否由说明书和附图充分支持。
- [ ] 并行布局发明专利与实用新型时，分别记录保护客体、共同结构基础、差异化范围、单一性和支持风险，不自动承诺双报。

## 5. 阶段门禁

- [ ] 核心连接、位置或运动关系缺失，导致方案无法复现或无法支撑必要特征时，停在发明挖掘或相应上游阶段并生成追问。
- [ ] 附图、附图标号或替代结构缺失但可与已批准权利要求支持分离时，按调用方核心 Skill 的合同记录缺口，不得擅自改变其 ready/blocked 判定。
- [ ] 领域检查不得放宽 `confirmed`、`source-backed`、审批、版本、新鲜度、输出数量或禁止编造规则。
